# 자바 계산기 프로젝트

`Generic`과 `enum`, 그리고 `Stream`을 활용하여 확장성을 높인 Java 기반 사칙연산 계산기 프로젝트입니다.

---

### 과제 내용

#### [STEP 1] 클래스 없이 기본적인 연산을 수행할 수 있는 계산기 만들기
> 계산기는 2개의 숫자를 받을 수 있고 사칙연산 될 문자를 받을 수 있다.
>
> 계산기는 exit을 입력할 때까지 계속해서 값을 받고 연산 결과를 반환한다.



#### [STEP 2] 클래스를 적용해 기본적인 연산을 수행할 수 있는 계산기 만들기
> 계산된 결과 값들을 기록하는 컬렉션을 만든다.
>
> 컬렉션의 가장 먼저 저장된 데이터를 삭제하는 기능을 만든다.


#### [STEP 3] Enum, 제네릭, 람다 & 스트림을 이해한 계산기 만들기
> 양의 정수만 받을 수 있었지만, 이제부터는 실수도 받을 수 있게 수정한다.
>
> 결과가 저장되어 있는 컬렉션을 조회하는 기능을 만든다.
> 그때 특정 값보다 큰 결과 값을 출력할 수 있도록 한다.

### 프로젝트 구조

>- `App.java` : 프로그램의 메인 루프를 담당하며, 전체적인 실행 흐름을 제어합니다.
>
>
>- `ArithmeticCalculator.java` : 실제 입출력 및 연산 로직, 결과 저장 및 관리, 메뉴 출력 기능이 담겨있습니다.
>
>
>- `OperatorType.java` : 상수와 연산자 기호를 매핑하고 검증하는 Enum 클래스입니다.

### 주요 기능

>- 다양한 숫자 타입 지원 : `Generic`을 활용하여 `Number`를 상속 받는 `Double` 등 다양한 숫자 타입 연산이 가능합니다.
>
>
>- Enum 연산자 관리 : 사칙연산 기호를 `OperatorType` `enum`으로 관리하였습니다.
>
>
>- 연산 기록 관리 : 연산 기록을 `List`에 저장하여 가장 오래된 계산 기록 삭제 및 계산 기록 조회를 할 수 있습니다.
>
>
>- Lambda 및 Stream 활용 : 저장된 기록들 중 입력 값보다 큰 결과들만 필터링하여 조회할 수 있습니다.

### 트러블 슈팅 및 고민 포인트

1. 예외 발생 시 흐름 제어
- 연산자 입력 오류 시에 이전에 저장된 연산자가 그대로 남아서 잘못된 계산이 수행되는 문제가 있었습니다.
- 해결 : `inputOperator()` 메서드가 `boolean`을 반환하게 하여, 입력 실패했을 시에 `App`에서 `continue`를 통해 다시 입력을 받도록 설계했습니다.

2. 프로그램 종료 로직
- `"exit"`를 입력했음에도 계산이 종료되지 않고 계속 수행되는 문제가 있었습니다.
- 해결 : `optionPrint()`의 반환 타입을 `boolean`으로 설정하여 사용자의 종료 의사를 메인 루프(while 문)에 전달, `break`가 작동하도록 구현했습니다.

### 향후 개선 사항

1. 입력 예외 처리 추가 : 숫자가 아닌 다른 타입을 입력했을 시 발생하는 로직 추가

2. 데이터 안정성 : 삭제할 기록이 없는 상태에서 삭제 시도 시 예외 처리
3. 반환값 명확화 : 연산 실패 시에 리턴되는 0.0이 실제 계산 값인지, 아니면 에러로 인한 기본 값인지 구현할 수 있도록 설계
